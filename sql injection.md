
### Title

**SQL Injection (Low Security) - DVWA Lab**

### Description

In this lab, I practiced SQL Injection on the low security level of the DVWA (Damn Vulnerable Web Application). The objective was to understand how unsanitized user input can be manipulated to interact with backend database queries.

During the exercise, I tested different SQL payloads to observe how user-supplied input can alter SQL statements and expose information from the database. This helped me understand the impact of improper input validation and the importance of secure coding practices.

### Key Learning Outcomes

* Understanding SQL Injection fundamentals
* Identifying vulnerable input fields
* Testing basic SQL payloads
* Observing database information disclosure
* Learning the importance of input validation and parameterized queries
* Understanding the security risks associated with SQL Injection

### Environment

* DVWA (Damn Vulnerable Web Application)
* Kali Linux
* Web Browser
